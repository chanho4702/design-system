# Chanho Design System — Phase 1 Wave 1 (Plan B1) 구현 계획

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Phase 0 파운데이션을 하드닝(typecheck·stylelint 강화·Button 회귀 테스트)하고, 토큰을 확장(green/amber)한 뒤, Radix가 필요 없는 정적 표시 컴포넌트 4개(Badge, Lozenge, Avatar, Spinner)를 추가한다.

**Architecture:** Phase 0과 동일 — 컴포넌트는 `packages/react/src/<Name>/` 4파일(tsx/module.css/test.tsx/stories.tsx) 패턴, 시맨틱 토큰(`--chanho-*`)만 참조, 스토리는 docs 앱이 자동 수집. 새 상태 컬러(success/warning)는 원시 팔레트 → 시맨틱 토큰 순서로 확장하며 light/dark 키 패리티 테스트가 자동으로 강제한다.

**Tech Stack:** Phase 0과 동일 (pnpm 11, React 19, CSS Modules, Vitest 3, Storybook 9.1, Stylelint 16) + `tsc --noEmit` typecheck

**참고:** 스펙 `design-system/docs/superpowers/specs/2026-07-05-design-system-design.md`, Phase 0 계획 `2026-07-06-phase0-foundation.md`
**후속:** Plan B2(Radix 도입 + 폼: TextField/Checkbox/Radio/Switch/Select), Plan B3(오버레이: Modal/Dropdown/Tooltip/Toast + Tabs)는 이 계획 완료 후 작성.

## Global Constraints

- 작업 루트: `C:\MSA_TEMPLATE\design-system\` (git 루트 `C:\MSA_TEMPLATE`, master 직접 커밋)
- 컴포넌트 CSS는 `--chanho-*` 변수만 사용. 허용되는 리터럴: 컴포넌트 고유 치수(width/height/border-width px), `line-height: 1`, 애니메이션 시간. 색상 리터럴(hex·색상명·rgb()·hsl() 등) 절대 금지 — Stylelint가 차단
- 시맨틱 컬러 토큰은 light/dark 동일 키 세트 (기존 패리티 테스트가 강제)
- 모든 컴포넌트: props TypeScript 완전 정의 + JSDoc, `ComponentPropsWithRef` 기반(React 19 ref-as-prop), className 병합 패턴은 Button과 동일 (`[styles.x, ..., className].filter(Boolean).join(" ")`)
- 새 컴포넌트는 TDD: 테스트 작성 → 실제 FAIL 관찰(RED) → 구현 → PASS(GREEN). RED/GREEN 실제 출력을 리포트에 기록
- 각 컴포넌트는 `src/index.ts`에 컴포넌트 + Props 타입 export 추가
- 커밋 컨벤션 `feat:` / `test:` / `chore:` / `docs:`
- 검증 명령: `pnpm typecheck`(Task 1 이후), `pnpm test`, `pnpm lint:css`, `pnpm build-storybook` (모두 design-system 루트에서)

```tsx
import type { ComponentPropsWithRef } from "react";
import styles from "./Badge.module.css";

export interface BadgeProps extends ComponentPropsWithRef<"span"> {
  /**
   * 시각적 스타일. brand는 강조 카운트, danger는 경고성 카운트에 사용한다.
   * @default 'neutral'
   */
  appearance?: "neutral" | "brand" | "danger";
}

export function Badge({ appearance = "neutral", className, ...rest }: BadgeProps) {
  const cls = [styles.badge, styles[appearance], className].filter(Boolean).join(" ");
  return <span className={cls} {...rest} />;
}
```

```tsx
import type { ComponentPropsWithRef } from "react";
import styles from "./Avatar.module.css";

export interface AvatarProps extends ComponentPropsWithRef<"span"> {
  /** 사용자 이름. 이미지 alt와 이니셜 생성에 사용한다. */
  name: string;
  /** 프로필 이미지 URL. 없으면 이니셜을 표시한다. */
  src?: string;
  /**
   * 크기(24/32/40px). 밀도 높은 목록에는 small을 사용한다.
   * @default 'medium'
   */
  size?: "small" | "medium" | "large";
}

function initialsOf(name: string): string {
  const words = name.trim().split(/\s+/).filter(Boolean);
  if (words.length === 0) return "";
  if (words.length === 1) return words[0].slice(0, 1).toUpperCase();
  return (words[0][0] + words[words.length - 1][0]).toUpperCase();
}

export function Avatar({ name, src, size = "medium", className, ...rest }: AvatarProps) {
  const cls = [styles.avatar, styles[size], className].filter(Boolean).join(" ");
  return (
    <span className={cls} {...rest}>
      {src ? <img className={styles.image} src={src} alt={name} /> : initialsOf(name)}
    </span>
  );
}
```
